## 二、逐文件详细变更

### 2.1 auth.py — 审计日志 + 拒绝协议确认机制

#### 新增审计日志系统

- 新增 `_audit_scrub()` — 净化审计元数据，剥离控制字符（防日志伪造/终端转义注入）
- 新增 `_audit()` — 结构化 JSON 行输出到 stderr，字段：`ts`、`op`、`success`、`meta`
- 新增 `_audit_logger` — 独立 logger，不传播到 root，避免干扰业务日志
- 覆盖操作：`send-code`、`verify`、`terms-check`、`terms-accept`、`terms-decline`、`status`
- 脱敏：手机号走 `_mask_phone`，userKey 走 `mask_user_key`

#### `_mask_phone` 短号码保护

- 长度 ≤ 7 的号码不再部分暴露，统一返回 `****`
- 空/None 也返回 `****`

#### `cmd_terms_decline` 增加确认机制

- 拒绝协议会清除本地 CXDA_USER_KEY / authtoken / authtoken_expire，属于敏感操作
- 三种确认方式（任一即可）：`--yes` / stdin `{"confirm":true}` / 交互终端输入 y/Y
- 未确认返回 `need_confirmation`，不动任何数据
- 新增命令行参数 `--yes` / `-y`

#### HTTP 请求加固

- `send-code` 和 `verify` 请求加 `allow_redirects=False`

### 2.2 common.py — 安全纵深加固

#### authtoken 加密存储

- `save_auth()` 现在同时加密 `CXDA_USER_KEY` 和 `authtoken`
- 读取时（`get_cached_token()`）透明解密 authtoken（兼容老明文格式）
- 缓解：authtoken 5 分钟有效期内仍是敏感凭证，明文缓存有泄漏风险

#### `_safe_env_executable` 支持多目录白名单

- `trusted_dir` 参数从单个 Path 改为支持 list/tuple
- 命中任一目录即通过

#### `CXDA_CACHE_PYTHON` RCE 防护加强

- 新增 `_python_trusted_dirs()` — Python 解释器可信目录白名单：
  - `/usr/bin`, `/usr/local/bin`, `/usr/sbin`, `/usr/local/sbin`
  - `/bin`, `/sbin`, `/opt/homebrew/bin`, `/usr/local/opt`, `/opt/python`, `/opt`
  - 动态加入 `sys.executable` 所在目录（venv/conda 等）
- `get_cache_python()` 调用 `_safe_env_executable` 时传入 `trusted_dir=_python_trusted_dirs()`
- 防止 `CXDA_CACHE_PYTHON=/tmp/evil_python` 环境变量注入执行任意解释器

#### `_is_valid_user_key` 格式校验

- 新增正则 `_USER_KEY_RE = ^[A-Za-z0-9_\-]{16,128}$`
- `get_user_key()` — 环境变量取值必须通过校验，否则忽略
- `save_user_key()` — 写入前强制校验，拒绝非法值
- 缓解：防止环境变量任意值绕过后端鉴权

#### 纯点名拒绝

- 新增 `_ALL_DOTS_RE = ^\.+$`
- `_validate_shared_filename()` 增加 `.`/`..`/`...` 拒绝
- 缓解：仅靠字符白名单不拦 `..`（不含分隔符仍可回溯父目录）

#### 新增 `http_post_form()`

- 统一 HTTP POST（application/x-www-form-urlencoded）
- 敏感字段走 POST body 而非 GET query string，避免落入 access log / Referer
- SSRF 校验、TLS 校验、超时策略与 `http_get` 一致
- `allow_redirects=False` 拒绝跟随重定向

#### `get_token()` 改为 POST

- 从 `requests.get(params={"userKey": ...})` 改为 `requests.post(data={"userKey": ...})`
- 加 `verify=True`, `allow_redirects=False`, `timeout=30`

#### SSRF 防护重构

- 新增 `_validate_official_url()` — 独立函数，scheme/host/path 三重校验
- 新增 `_decode_response()` — 统一 gzip+base64 解码，增加 32MiB 解压上限
- `http_get()` 加 `verify=True`, `allow_redirects=False`

### 2.3 cred_crypto.py — 密钥派生大幅加固

#### 引入 pepper 文件

- 新增 `_get_or_create_pepper()` — 32 字节 secure random，存于 `~/.cxda-cache/.cred_pepper.bin`
- 权限 0o600，O_NOFOLLOW + O_EXCL 首次创建
- 读取时 lstat 校验：拒绝符号链接、拒绝非普通文件
- 缓解：同机其他用户即使知道 host/user/machine-id，因无法读 pepper 而无法派生密钥

#### 用户名获取不再信任环境变量

- 旧版：`getpass.getuser()` → 受 LOGNAME/USER 环境变量影响
- 新版：`_get_effective_username()`
  - POSIX：`pwd.getpwuid(os.getuid())`，直接从口令数据库按 UID 反查
  - Windows：`GetUserNameW` WinAPI，从访问令牌读取
  - 杜绝环境变量污染

#### 密钥材料改用 JSON 序列化

- 旧版：`f"{host}|{user}|{machine_id}"` — 分隔符碰撞风险
- 新版：`json.dumps({"host":..., "user":..., "machine_id":...}, sort_keys=True, separators=(",",":"))`
- JSON 转义 `"` 和 `\`，保证不同输入必然对应不同序列化结果
- pepper 独立成段附加：`material = identity + b"\x00pepper=" + pepper`

#### 三项特征均不可空

- 旧版仅要求 host+user 非空
- 新版要求 host+user+machine_id 全部非空，任一缺失拒绝派生

#### 解密异常细分

- `InvalidToken` / `ValueError` → 返回 `("", False)`
- 其他 `Exception` → 也返回 `("", False)`（旧版吞掉异常）

### 2.4 cxda_cache_cli.py — symlink 攻击防护

#### 新增 `_ensure_dir_not_symlink()`

- 检测目录是否为符号链接，拒绝 `ln -s /etc ~/.cxda-cache` 这类攻击
- 也拒绝非目录节点
- 默认回退路径同样走检测

#### `/root` 从关键目录黑名单移除

- root 用户家目录就是 /root，黑名单导致 root 永远拿不到工作空间
- /root 的访问隔离由文件系统权限（0o700）+ 本模块 0o600 保证

#### 新增 `_secure_read_text()`

- O_NOFOLLOW 读取文本文件，缓解 symlink race（TOCTOU）
- 替代 `Path.read_text()`（会跟随符号链接）

#### `_secure_write_text()` 加 O_NOFOLLOW

- 写入也拒绝符号链接

#### delete 操作改用词法路径

- 新增 `_get_shared_path_lexical()` / `_get_file_path_lexical()` — 不 resolve
- delete 只删软链本身，不跟随目标
- 用 `os.lstat` 判断类型，`os.unlink` 直接操作

#### 文件名/技能名校验增强

- 公域：新增 `_validate_shared_filename()` — 字符白名单 + 纯点名拒绝
- 私域：新增 `_validate_private_filename()` — 同上
- skill_name：`_validate_skill_name()` 增加纯点名拒绝

---

### 2.5 data_fetcher.py — 输入校验 + 日志净化

#### `normalize_stock_code` 不再兜底

- 旧版：未识别格式默认当 A 股（`return 'a', code`）
- 新版：返回 `(None, None)`，调用方拒绝
- 缓解：兜底会把任意字符串作为 stkCode 传给后端 API，可能产生 SQLi/注入
- 港股数字部分增加长度校验：`1 <= len(numeric_part) <= 5`
- 新增类型检查：`not isinstance(stock_code, str)` → 返回 `(None, None)`

#### 新增 `_sanitize_for_log()`

- 剥离控制字符（`\r\n\t\x00-\x1f\x7f`），防止日志注入/伪造审计条目
- 所有 logger 输出改用 `_sanitize_for_log()` 包裹

#### 非法股票代码提前拒绝

- `market is None` 时 logger.warning 并返回 None

---

### 2.6 notifier.py — 输出安全加固

#### `_sanitize` 增加单引号转义

- 新增 `'` → `&#x27;`，缓解 HTML 属性上下文 XSS（如 `<div title='{value}'>`）

#### 新增 `_safe_int_score()`

- 评分类字段强转为 int，限制 [0, 100] 范围
- 非法值回退 default=50
- 缓解：AI/API 返回的 sentiment_score 若为字符串或含控制字符

#### 新增 `_fmt_num()`

- 数值字段强制 float 转换后再格式化
- 非法值输出 `N/A`
- 缓解：价格/均线/乖离率/支撑位/压力位若为不可信字符串

#### 输出改用安全函数

- `sentiment_score` → `_safe_int_score()`
- 当前价格/MA5/MA10/MA20/乖离率/支撑位/压力位 → `_fmt_num()`
- 去除 `isinstance(tech['ma5'], (int, float))` 前置检查，统一走 `_fmt_num`

### 2.7 query.py — 跟随 common.py 改动

- `import http_post_form`
- `get_token()` 和 `query_api()` 中 `http_get(params=...)` → `http_post_form(data=...)`
- userKey 从 GET query string 改为 POST body
